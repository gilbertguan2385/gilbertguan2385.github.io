Git

发表于 2019-05-30 | 分类于 工具

简介

Git是Linux之父Linux Torvalds开源的目前最好用的分布式版本控制系统，没有之一。

基本概念

  • 工作区 working dir
    版本控制范围的文件夹及文件，即.git文件夹同级的其他文件夹及文件

  • 暂存区 stage

    未被Git版本管理的文件[unstage]到纳入Git版本管理的文件[committed]之间的过渡，不可跳过，新增之后的文件add后会放到暂存区，reset head后回到unstage状态

常用命令

  • 命令帮助

    git help command

    git help -a

    git help -g

  • 初始化库

    git init

  • 添加文件至暂存

    git add filename

  • 清空未跟踪的文件

    git clean -nfd

    -n 参数查看会删掉哪些文件，防止重要文件被误删

  • 撤销工作区的修改

    git checkout – filename

  • 提交到库

    git commit -m “desc”

  • 查看状态

    git status

  • 文件比较

    git diff filename

  • 文件比较(最新版本与工作区)

    git diff HEAD – filename

  • 提交日志

    git log
    git log –pretty=oneline
    git log –graph –pretty=oneline -abbrev-commit

  • 版本回退：
    git reset –hard/–soft HEAD^
    git reset –hard/–soft HEAD~number
    git reset – hard/–soft commit_id

  • 版本文件删除：
    git rm -r filename

  • 查看命令历史：
    git reflog –pretty=oneline

远程库

  • 查看远程库：
    git remote git remote -v

  • 关联远程库：
    git remote add remote_repo git@*.com:remote_repo

    eg:
    git remote add origin git@github.com:michaelliao/learngit.git

  • 推送到远程：
    git push -u remote_repo local_branch 第一次推送本地分支的所有内容
    git push remote_repo local_branch

  • 从远程更新
    git fetch
    git fetch -t -p -f
    git pull

  • 从远程库克隆：
    Git clone
    eg:
    git clone git@github.com:michaelliao/gitskills.git

分支管理

  • 查看分支
    git branch git branch -a

  • 创建分支
    git branch branch_name

  • 删除分支
    git branch -d branch_name
    git branch -D branch_name 强制删除未被合并的分支

  • 切换分支
    git checkout branch_name

  • 创建并切换分支
    git checkout -b branch_name

  • 合并分支
    git merge Fast-forward模式，看不出有合并过，删除分支后会丢掉分支信息
    git merge –no-ff -m “desc” other_branch 禁用Fast-forward模式，可看出合并

  • 本地分支与远程分支关联
    git branch –set-upstream local_branch remote_branch

Bug分支

  • 隐藏工作区
    git stash

  • 查看隐藏的工作区
    git stash list

  • 恢复工作区
    git stash apply

  • 查看已经删除的工作区
    git fsck –unreachable
    git fsck –lost-found
    git fsck –lost-found|grep ‘ ‘|awk ‘{print $3}’

    恢复已经丢失的commit：
    git merge commit_id

  • 删除stash内容
    git stash drop

  • 恢复并删除
    git stash pop

  • 重新生成change_id :
    git commit –amend

  • 推送分支到远程Gerrit:
    git push origin :refs/for/

标签管理

  • 创建标签
    git tag tag_name
    git tag tag_name commit_id

  • 创建制定标签名的标签
    git tag -a tag_name -m “desc” commit_id

  • 查看标签 git tag
    查看标签信息 git tag tag_name

  • 删除标签
    git tag -d tag_name

  • 删除推送远程的标签
    git push remote_repo :refs/tags/tag_name

  • 推送标签到远程
    git push remote_repo tag_name
    git push remote_repo –tags

  • 配置别名
    git config –list|grep ‘alias’
    git config –global alias.name command
    eg: git config –global alias.last log -1

  • 已配置别名
    vi ~/.gitconfig

    [user]
    name = user_name
    email = user_name@domain

    [alias]
    alias.logp=log –graph –pretty=oneline –abbrev-commit

    alias.alias=!git config –list | grep ‘alias’

    alias.push=push origin :refs/for/

    alias.ck=checkout

    alias.merge=merge –no-ff

    alias.coamend=commit –amend

    alias.cls=’clear’

    alias lostandfound=’’'‘//*/gitLostAndFound.sh’'‘’

    根据提交记录中的关键字获取丢失的commit_id
